# 使用方法

本文档说明如何在 Unity 编辑器中使用已实现的代码框架来搭建游戏。

## 1. 前置准备

### 1.1 创建 Scene

需要在 `Assets/Scenes/` 下创建以下 Scene，并将它们全部添加到 **Build Settings** 的 Scene 列表中：

| Scene 名称 | 用途 |
|------------|------|
| `LevelSelect` | 选关页面 |
| 任意关卡 Scene（如 `Level_Forest`） | 关卡场景 |

关卡场景名不再要求按数字顺序命名；只要关卡按钮对象的 `GameObject.name` 与场景名一致即可。

### 1.2 创建节点 Prefab

`Node` 是一个抽象类，不能直接使用。需要创建至少一个具体的节点子类，例如：

```csharp
public class BasicNode : Node
{
    // 最简单的节点，无需额外逻辑
}
```

每个节点 Prefab 需要包含以下组件：

| 组件 | 说明 |
|------|------|
| **Node 子类** | 如 `BasicNode`，设置 `CanRotate`、`CanCollide` 等节点行为字段 |
| **NodeDragHandler** | 自动处理建造模式下的拖拽 |
| **NodeRenderer** | 控制节点颜色（正常/仓库半透明） |
| **SpriteRenderer** | 节点的视觉表现（圆形/方形/三角形 Sprite） |
| **Collider2D** | 碰撞箱（CircleCollider2D 或 PolygonCollider2D） |
| **Rigidbody2D** | 会由代码自动添加（如果没有的话） |

## 2. 搭建选关场景（LevelSelect）

1. 打开 `LevelSelect` Scene
2. 创建一个空 GameObject，命名任意（如 `LevelSelectUI`）
3. 添加 `LevelSelectManager` 组件
4. 在场景中手动摆放关卡按钮对象，每个按钮对象挂 `AutoLevelButton`：
   - `GameObject.name`：对应关卡 Scene 名称
   - `UnlockLevels`：本关通关后解锁的关卡按钮列表
   - 若对象使用 `GameButton`（UI 模式），会走 `GameButton` 点击和着色；
   - 若对象不使用 `GameButton`（场景物体模式），请保证有 `Collider`，点击通过 `OnMouseDown` 触发，颜色会写到 `SpriteRenderer/Renderer`。
5. 可选：添加 `LevelSelectTopologyRenderer`，用于显示按钮到 `UnlockLevels` 的白色连线。
6. `LevelSelectManager` 仅需配置 `QuitButtonSize`，退出按钮仍会自动生成。
7. `AutoLevelButton` 会在编辑器中通过 Gizmos 绘制到 `UnlockLevels` 的连线，便于检查拓扑关系。

解锁规则：

- 若一个关卡没有任何父关卡（即没有其他按钮把它放进 `UnlockLevels`），则自动解锁；
- 若一个关卡有父关卡，则必须其所有父关卡都已通关才解锁（与逻辑）；
- 其它情况为锁定状态。

关卡按钮状态材质：

- `ColorConfig.CompletedLevelMaterial`：已通关材质；
- `ColorConfig.UnlockedLevelMaterial`：已解锁材质；
- `ColorConfig.LockedLevelMaterial`：未解锁材质。

若对应材质未配置，则会回退到旧的颜色字段（`CompletedLevelColor`、`UnlockedLevelColor`、`LockedLevelColor`）。

## 3. 搭建关卡场景（Level_X）

### 3.1 必需的 Scene 内对象

每个关卡场景需要手动放置以下对象：

#### 地面

- 创建 GameObject，添加任意 2D Collider（如 PolygonCollider2D、BoxCollider2D）
- 添加 Rigidbody2D，设置 Body Type = **Static**
- 添加 SpriteRenderer 进行视觉表现

#### 自动地面（AutoGround）

- 创建一个根 `Empty`，挂上 `AutoGround` 组件；
- `AutoGround` 会自动创建：
  - `GroundRoot`（子 Empty，用来挂地面实例）
  - `PointA` / `PointB`（两个端点 Empty）；
- 给 `Ground Prefab` 指定 `Assets/Prefabs/地图/地面.prefab`；
- 将 `PointA`、`PointB` 拖到目标位置后，地面实例会自动更新：
  - `position.xy` = 两端点中点
  - `rotation.z` = 两端点连线角度
  - `localScale.x` = 两端点距离 / `LengthPerScaleX`
- 更新策略：
  - 编辑器模式：保持实时更新，便于关卡搭建
  - Play 模式：仅当端点位置或关键参数变化时更新，避免每帧重建碰撞体

建议让地面 prefab 的默认长度对应 `LengthPerScaleX = 1`，这样更直观。

#### 自动弧形地面（AutoArcGround）

- 创建一个根 `Empty`，挂上 `AutoArcGround` 组件；
- `AutoArcGround` 会自动创建：
  - `GroundRoot`
  - `PointA` / `PointMiddle` / `PointB`；
- 弧形地面会在运行时按三点生成：
  - 圆弧必须经过 `PointA`、`PointMiddle`、`PointB`
  - 不依赖地面 prefab，直接生成 Mesh 和 `PolygonCollider2D`
  - 自动挂 `Rigidbody2D(Static)`
- 可在 Inspector 调整：
  - `ArcThickness`：弧形地面厚度
  - `SegmentCount`：圆弧分段数（越大越平滑）
  - `DoubleSidedGap`：双面间距（沿本地 Z 方向分离前后面，避免重合闪烁）
  - `GroundMaterial`：生成 Mesh 使用的材质
- 若三点共线（无法唯一确定圆），该弧形地面会自动隐藏并禁用碰撞体。
- 更新策略：
  - 编辑器模式：保持实时重建，便于调形
  - Play 模式：仅当三点位置或关键参数变化时重建，降低运行态碰撞体抖动

#### 终点

- 创建 GameObject，添加 2D Collider
- **勾选 Collider 的 Is Trigger**
- 添加 `GoalTrigger` 组件
- 添加 SpriteRenderer 进行视觉表现
- 终点材质由 `ColorConfig` 控制：
  - `GoalDefaultMaterial`：初始材质
  - `GoalReachedMaterial`：被任意节点触发后的材质

#### 建造区域标记

- 创建一个空 GameObject 作为建造区域中心（如 `BuildAreaCenter`）
- 放到地图中合适的位置

#### 相机锚点

- 创建一个空 GameObject 作为建造模式相机位置（如 `CameraAnchor`）
- 其 position 的 x、y 决定建造时相机中心，z 保留默认值
- 可选：创建一个空 GameObject 作为关卡入场相机位置（如 `IntroCameraAnchor`）
- 入场时相机会先停留在 `IntroCameraAnchor`，随后平滑移动到 `CameraAnchor`
- 若不设置 `IntroCameraAnchor`，系统会自动根据场景内可见物体计算入场相机中心与缩放，确保视野包围整体物体
- 若希望手动指定该关卡的全图缩放，可在 `LevelManager.IntroCameraOrthoSizeOverride` 填入大于 0 的值（会同时影响入场、悬浮预览和青色 Gizmo 边框）。

### 3.2 LevelManager

1. 创建一个空 GameObject（如 `LevelManager`）
2. 添加 `LevelManager` 组件
3. 设置 Inspector 字段：

| 字段 | 说明 |
|------|------|
| Intro Camera Anchor | 关卡入场相机锚点（全图预览） |
| Intro Camera Ortho Size Override | 关卡全图视角缩放覆盖值（<=0 表示使用自动/默认） |
| Build Area Center | 拖入上面创建的建造区域中心 Transform |
| Build Area Size | 建造区域的宽高 |
| Camera Anchor | 拖入相机锚点 Transform |
| Inventory Area | 仓库区域的 Rect（x, y, width, height，世界坐标） |
| Initial Blueprint | 初始蓝图（见下方说明） |
| UI Button Size | UI 按钮大小 |
| 各颜色字段 | 退出 / 开始 / 中止 / 下一关按钮的颜色 |

#### 配置初始蓝图（Initial Blueprint）

初始蓝图决定了关卡中可用的节点类型和数量。配置步骤：

1. 展开 `Initial Blueprint` > `Nodes` 列表
2. 点击 `+` 添加条目，每个条目代表一个节点
3. 为每个条目设置 `Prefab` 字段（拖入对应的节点 Prefab）
4. `Is In Inventory` 默认为 `true`（初始时所有节点都在仓库中）
5. 其他字段（`posX`、`posY`）无需手动填写，运行时自动处理
6. `NodeData.nodeType` 已弃用；蓝图恢复仅依赖 `Prefab`，若 `Prefab` 为空会直接报错并跳过该节点
7. 蓝图会保留节点实例的来源 Prefab（包含同类型不同参数的变体），不会再按类型名回退匹配
8. 蓝图恢复后会对所有场上节点按当前位置重新执行一次自动放置连接，用当前连接规则刷新关系

例如：一个关卡需要 3 个 BasicNode 和 2 个 WheelNode，则在列表中添加 5 个条目，前 3 个设置 BasicNode Prefab，后 2 个设置 WheelNode Prefab。

#### 蓝图重置

在建造模式下按 **R 键** 可重置蓝图为初始状态（删除磁盘存档，所有节点回到仓库），并重新播放入场相机与零件出现动画。

### 3.3 ConnectionRenderer

1. 创建一个空 GameObject（如 `ConnectionRenderer`）
2. 添加 `ConnectionRenderer` 组件
3. 可选：设置 `Line Width` 与 `Line Material`
4. 连接线颜色由 `ConnectionColorConfig` 控制：在 `Assets/Resources` 中创建 `ConnectionColorConfig`（Create > Game > Connection Color Config），并配置：
   - `ConnectionColor`
   - `ConnectionWidth`
   - `ChargedConnectionColor`
   - `ChargedConnectionWidth`
   - `BalloonConnectionColor`（气球连接专用颜色）
   - `BalloonConnectionWidth`
   - `LevelConnectionLockedColor`（选关拓扑连接：前置关未通关）
   - `LevelConnectionLockedWidth`
   - `LevelConnectionUnlockedColor`（选关拓扑连接：前置关已通关）
   - `LevelConnectionUnlockedWidth`
   - `LevelConnectionZ`（选关拓扑连接统一使用的 Z 值，默认 1000）
   - `PreviewColor`

### 3.4 可建造区域显示配置

可建造区域和仓库区域的填充色与边框由 `BuildAreaConfig` 控制。请在 `Assets/Resources` 中创建 `BuildAreaConfig`（Create > Game > Build Area Config），并配置以下六个参数：

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `BuildAreaFillColor` | 黑色透明 | 可建造区域填充颜色 |
| `BuildAreaBorderColor` | 白色 | 可建造区域边框颜色 |
| `BuildAreaBorderWidth` | 0.06 | 可建造区域边框粗细 |
| `InventoryAreaFillColor` | 黑色透明 | 仓库区域填充颜色 |
| `InventoryAreaBorderColor` | 白色 | 仓库区域边框颜色 |
| `InventoryAreaBorderWidth` | 0.06 | 仓库区域边框粗细 |

### 3.5 相机

1. 选中 Main Camera
2. 添加 `RuntimeCameraController` 组件
3. 设置参数：

| 字段 | 说明 | 默认值 |
|------|------|--------|
| Smooth Time | 相机平滑跟随的时间常数 | 0.3 |
| Min Ortho Size | 最小缩放 | 3 |

`RuntimeCameraController` 在建造模式下自动禁用（由 LevelManager 控制）。
运行模式默认使用“只缩小不放大”缩放模式（`ZoomMode.OnlyZoomOut`）：镜头可继续拉远，但不会自动拉近。
运行/通关阶段悬浮预览全图时，退出预览后会直接恢复跟随；并在冷却阶段暂时禁用 `OnlyZoomOut` 限制与“基于当前相机位置的放缩放大项”，时长为 `RuntimeCameraController.SmoothTime * 5`，以降低切换抖动。

入场动画参数由 `CameraConfig` 控制。请在 `Assets/Resources` 下创建 `CameraConfig`（Create > Game > Camera Config）并配置：

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `IntroHoldSeconds` | 1 | 入场全图停留时长 |
| `IntroMoveSeconds` | 3 | 相机移动到组装位时长 |
| `IntroBoundsScaleMultiplier` | 1.2 | 自动取景时在黄色包围盒基础上额外放大倍率 |
| `HoverToIntroSeconds` | 0.35 | 建造阶段悬浮预览按钮时，镜头切到开场视角的过渡时长 |
| `HoverReturnSeconds` | 0.2 | 悬浮结束后镜头快速回到建造位的过渡时长 |
| `RuntimeHoverToIntroSeconds` | 0.35 | 运行/通关阶段悬浮预览按钮时，镜头切到全图视角的过渡时长 |
| `PartAppearSeconds` | 0.5 | 库存零件出现动画时长 |
| `PartAppearStartScale` | 0.65 | 出现动画初始缩放比例 |
| `PartAppearStartAlpha` | 0 | 出现动画初始透明度比例 |

## 4. 创建自定义节点类型

继承 `Node` 类来创建不同类型的节点：

```csharp
public class WheelNode : Node
{
    protected override void Awake()
    {
        CanRotate = true;
        CanCollide = true;
        base.Awake();
    }
}
```

```csharp
public class GhostNode : Node
{
    protected override void Awake()
    {
        CanRotate = false;
        CanCollide = false;
        base.Awake();
    }
}
```

```csharp
public class ClickableNode : Node
{
    public override void OnRuntimeClick()
    {
        // 运行时点击效果，如施加向上的力
        Rb.AddForce(Vector2.up * 500f);
    }
}
```

### 节点属性说明

| 属性 | 含义 |
|------|------|
| `CanCollide = true` | 有物理碰撞 |
| `CanCollide = false` | 不与其它物体发生物理碰撞，但仍可触发终点 |
| `CanRotate = true` | 参与旋转，连接会产生角向弹簧力矩 |
| `CanRotate = false` | 不旋转（碰撞箱是光滑圆或不碰撞） |
| `CanCharge = true` | 可充电源，始终可导电，可为连接网络供电 |
| `CanConduct = true` | 可导电，可从已带电节点获得充电 |

电学规则：

- 可充电节点总是可导电；
- 可导电节点必须与可充电节点直接连接才会被充电；
- 被充电节点不会继续向外中继充电；
- 当且仅当一条连接满足“其中一端是可充电节点，另一端是被充电节点”时，该连接显示为 `ConnectionColorConfig.ChargedConnectionColor`（默认绿色）。

气球连接规则：

- `BalloonNode` 不会与 `BalloonNode` 建立连接；
- `BalloonNode` 与其他节点可建立连接，但该连接为“气球连接”；
- 气球连接只保留径向弹簧力，不会在两端交换力矩；
- 气球连接颜色使用 `ConnectionColorConfig.BalloonConnectionColor`。

## 5. 调整物理参数

连接参数与全局物理参数由 `NodeConfig` 控制。请在 `Assets/Resources` 下创建 `NodeConfig` 资产（Create > Game > Node Config）并在 Inspector 中修改：

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `MaxConnectRadius` | 3 | 节点可连接最大距离 |
| `MinConnectRadius` | 2 | 节点可连接最小距离 |
| `MaxConnectNumber` | 10 | 单节点最大连接数 |
| `SpringK` | 125 | 弹簧刚度（越大越硬） |
| `SpringDamping` | 5 | 弹簧阻尼（防止抽搐） |
| `SpringBreakLength` | 1 | 弹簧断裂阈值（形变超过此值则连接断裂） |
| `AngularSpringK` | 200 | 角向弹簧刚度 |
| `AngularSpringDamping` | 1 | 角向弹簧阻尼 |

## 6. 存档说明

存档文件保存在 `Application.persistentDataPath` 下：

| 文件 | 内容 |
|------|------|
| `save.json` | 已通关关卡的 Scene 名称列表（例如 `Level_Forest`） |
| `blueprint_<SceneName>.json` | 各关卡的建造蓝图（按场景名区分） |

删除这些文件即可重置存档。
当前版本仅按 `SceneName` 读写蓝图，不再使用旧的 `blueprint_<index>.json` 回退逻辑。
若某关卡蓝图文件读取/反序列化异常，系统会将该蓝图视为不存在，并自动回退为 `Initial Blueprint`。

`persistentDataPath` 的典型路径：
- Windows: `C:/Users/<用户名>/AppData/LocalLow/<公司名>/<项目名>/`
- macOS: `~/Library/Application Support/<公司名>/<项目名>/`

## 7. 运行流程总结

```
启动游戏
  └─ 选关页面（LevelSelect Scene）
       ├─ 点击已解锁关卡 → 进入关卡
       └─ 点击退出按钮 → 退出游戏

关卡内部（Level_X Scene）
  └─ 入场引导
       ├─ 相机在入场锚点停留（默认 1s）
       ├─ 相机平滑移动到组装相机位（默认 3s）
       ├─ 建造区/仓库区背景与边框淡入显示
       ├─ 入场阶段仓库零件不显示
       ├─ 仓库零件淡入+缩放出现（默认 0.5s）
       └─ 动画期间仅允许 Exit，结束后开放零件拖拽/旋转
  └─ 建造模式
       ├─ 从仓库拖拽节点到建造区域
       ├─ 节点自动连接（半径内最近的节点）
       ├─ 可不放完全部节点；仅要求“场上节点全部连通”即可开始运行
      ├─ 右上预览按钮：鼠标悬浮可预览开场初始视角（图像为四角三角形，点击区域仍是原方形区域）
       ├─ 悬浮预览期间：隐藏建造/仓库区域与仓库零件，仅允许 Exit 与悬浮本身
       ├─ 鼠标移出后：镜头快速回到原位，区域与仓库零件淡入恢复，并恢复交互
       ├─ 按 R 键 → 重置为初始蓝图（所有节点回到仓库）
       ├─ 点击右下圆形按钮 → 开始运行
       └─ 点击左上三角按钮 → 保存蓝图，返回选关
  └─ 运行模式
       ├─ 点击开始后会在下一个 FixedUpdate 进入运行态
       ├─ 节点获得物理，开始运动
       ├─ 相机跟随质心
       ├─ 点击右下方形按钮 → 中止，恢复蓝图回到建造
       └─ 所有终点都至少被碰到一次 → 过关
  └─ 过关模式
       ├─ 物理继续运行
       ├─ 自动保存通关记录
      └─ 点击右下三角按钮 → 直接读取 `UnlockLevels[0]` 作为下一关；若 `UnlockLevels` 为空、`UnlockLevels[0]` 为 null/空，或该关未解锁，则返回选关
```

## 8. 注意事项

- 所有关卡 Scene 和 `LevelSelect` Scene 必须加入 **File > Build Settings** 的 Scene 列表中，否则 `SceneManager.LoadScene` 会失败。
- 每个关卡 Scene 中的地面和终点需手动搭建，代码不会自动生成。
- 节点 Prefab 上不需要手动添加 Rigidbody2D，`Node.Awake()` 会自动创建。
- 美术规范：所有元素只用圆形、三角形、长方形表达，不允许文字和数字。
- 调试快捷键（选关界面）：`Ctrl + T` 会将当前拓扑中已注册的全部关卡写入通关存档。
- 关卡入场动画期间会锁定交互（仅放行 Exit 按钮），这是预期行为。
- 建造阶段悬浮预览期间同样会锁定大部分交互，并临时隐藏建造/仓库可视元素，这是预期行为。
